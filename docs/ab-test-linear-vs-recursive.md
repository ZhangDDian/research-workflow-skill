# A/B 测试：线性 vs 递归搜索模式

> 测试日期：2026-05-27
> 测试任务：调研 AI 自主代理领域 2025-2026 年开源方案，评估 claude-avatar 演进方向
> 控制变量：同一任务描述、同一模型（Opus）、同一子代理类型（general-purpose）

---

## 一、两种模式定义

### 线性模式

预设关键词 → 按固定步骤执行 → 不回头。

```
Step 1: SCAN   — 用 5 个预设关键词搜索（关键词在 prompt 中写死）
Step 2: FILTER — 从结果中筛 Top 5
Step 3: COMPARE — 做对比表
Step 4: RECOMMEND — 给建议
Step 5: 写报告
```

核心约束：
- 搜索只在 Step 1 做，后面不再搜
- 不允许跳步或修改之前结论
- 关键词由 prompt 作者在发送前确定

### 递归模式

假设驱动 → 每轮评估 → 基于评估决定下一轮方向。

```
Phase 1: 理解与假设（1 次）
  - 列初始假设和不确定项清单

Phase 2: 递归搜索（最多 5 轮）
  每轮 4 步：
  a. 搜索 — 针对当前最大不确定项
  b. 提取 — 关键事实
  c. 状态评估 — 新事实 / 假设更新 / 意外
  d. 方向决策 — 基于此刻认知决定下一轮搜什么

  终止条件：连续 2 轮无新事实 OR 5 轮上限 OR 全部解决

Phase 3: 报告（1 次）
```

核心约束：
- 不预设搜索关键词
- 允许中途修正理解
- 允许追踪意外发现

## 二、效率对比

| 指标 | 线性 | 递归 | 倍数 |
|------|------|------|------|
| Token 消耗 | 29K | 96K | 3.3x |
| 工具调用 | 11 次 | 37 次 | 3.4x |
| 耗时 | 159s | 913s | 5.7x |
| 搜索轮次 | 5（预设） | 5（自适应） | 1x |

递归模式贵 3 倍、慢 6 倍。

## 三、覆盖范围对比

| 维度 | 线性 | 递归 | 递归独有 |
|------|------|------|---------|
| 主流框架 | AutoGPT, CrewAI, LangGraph, Agno, EvoAgentX | LangGraph, CrewAI, Google ADK, OpenAI SDK, Mastra, Dify | Mastra（4层记忆）, Dify, ADK |
| 记忆方案 | EvoAgentX, 泛泛提及向量库 | Letta, Zep/Graphiti, Mem0, LangMem, Supermemory | 5 个专业记忆框架的深度对比 |

线性覆盖 5 个方案，递归覆盖 15+ 个方案/项目/论文。

## 四、产出质量对比

| 维度 | 线性 | 递归 |
|------|------|------|
| 结论结构 | 3 方向（记忆/目标/可观测性），偏泛 | 12 项改进 × 3 阶段路线图 + 优先级矩阵，可直接执行 |

| 认知演化 | 无（一次性输出） | 完整记录 5 轮假设变化过程 |
| 代码示例 | 无 | 每个建议配 Python/JSON/XML 代码片段 |
| 核心判断 | "坚持极简，纵向深挖"（正确但笼统） | "差距不在哲学在工程成熟度"+ 4 项具体差距（精准） |

## 五、关键差异分析

### 线性漏掉了什么

1. **没找到 Nightcrawler** — 一个与调研对象几乎同构的项目。预设关键词没命中它
2. **没查 Anthropic 官方 harness 最佳实践** — 最权威的参考，但线性模式没有"追踪意外"的能力
3. **记忆方案停留在表面** — 只到"应该用向量库"的层面，没深入到 Letta/Mem0/Supermemory 的具体对比
4. **建议可操作性弱** — "引入记忆蒸馏"没有代码、没有路径

### 递归做对了什么

1. **第 2 轮追踪第 1 轮的意外**：发现 Claude Agent SDK 后立刻深挖，产生了"从 subprocess 迁移到 SDK"这个高价值建议
2. **第 3 轮补短板**：前两轮暴露记忆是盲区，专门搜记忆框架，找到 5 个专业方案
3. **第 5 轮跨域碰撞**：Nightcrawler + Anthropic harness + 自我改进三者交叉，产生了"不可变任务追踪"和"98.4/1.6 法则"两个最有价值的洞察

## 六、适用场景

| | 线性 | 递归 |
|---|---|---|
| 适合 | 快速概览、已知领域竞品对比、时间/预算有限 | 未知领域深度探索、需要可执行建议、重要决策 |
| 不适合 | 需要发现"不知道自己不知道的"东西 | 简单信息查询、已有明确方向 |

---

## 七、核心结论

递归模式的优势不是"搜得更多"，而是**能追踪意外**。

每轮的状态评估（新事实 / 假设更新 / 意外）让模型可以临时改变方向去追一个意外发现。线性模式在搜索开始前就锁定了所有关键词——在信息最少的时刻做了最多的决策。

这验证了涌现方法论的判断：**计划在信息最少的时刻锁定路径，递归在每个节点用当下全部认知重新决策**。

### 实际采用建议

调研工作流 v1.4.0 已将 researcher.md 的执行流程从线性改为递归。同时保留线性模式的优点：

- W1-W6 工作流模板从"必须执行的步骤"降级为"质量检查清单"（确保覆盖该覆盖的维度，但不锁定顺序）
- 报告骨架仍然是固定结构（防止自由发挥导致结构混乱）
- 5 轮硬上限防止无限发散

## 附录：A/B 测试 Prompt

### 线性模式 Prompt

```
你是一个技术调研专家。请按以下**线性流程**严格按顺序执行，不要跳步，不要回头。

## 调研任务
调研 AI 自主代理（autonomous agent）领域 2025-2026 年的最新开源方案和技术趋势，
评估一个叫 claude-avatar 的项目的下一步演进方向。

[claude-avatar 项目描述，两组相同]

## 执行流程（严格线性，按顺序执行）

### Step 1: SCAN（搜索）
用 WebSearch 搜索以下关键词，每个搜 1 次：
1. "autonomous AI agent framework 2025 2026 open source"
2. "AI agent self-evolving memory persistent"
3. "claude code autonomous agent"
4. "AI digital twin autonomous workflow"
5. "LLM agent loop memory architecture"

### Step 2: FILTER（过滤）
从搜索结果中筛选 Top 5 最相关的项目/方案

### Step 3: COMPARE（对比）
claude-avatar vs 这 5 个方案对比表

### Step 4: RECOMMEND（推荐）
3 个演进方向建议

### Step 5: 写报告

注意：
- 严格按 Step 1→2→3→4→5 顺序执行，不要跳步
- 不要在中途回头修改之前步骤的结论
- 搜索只在 Step 1 做，后面步骤不再搜索
```

```

## 调研任务
[同上]

### Phase 1: 理解与假设（做 1 次）
1. 分析核心设计和当前能力边界
2. 列出初始假设
3. 列出不确定项清单

### Phase 2: 递归搜索（最多 5 轮）
每轮执行 4 步：
a. 搜索 — 针对当前最大不确定项
b. 提取 — 关键事实
c. 状态评估 — 新事实 / 假设更新 / 意外
d. 方向决策 — 基于评估决定下一轮搜什么

终止条件：连续 2 轮无新事实 OR 5 轮上限 OR 全部解决

### Phase 3: 报告（做 1 次）

注意：
- 不要预设搜索关键词列表——每轮方向由上一轮评估决定
- 允许中途修正对问题的理解
- 允许追踪意外发现
```
