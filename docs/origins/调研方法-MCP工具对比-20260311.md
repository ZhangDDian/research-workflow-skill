# 调研报告: AI 辅助调研工具与 MCP Server 对比

**日期**: 2026-03-11
**任务**: 对比 Claude Code / AI agent 用户实际使用的调研工具：Context7、Tavily、Exa、Brave Search、Perplexity、Firecrawl、内置 WebSearch、Baidu Search MCP

---

## 调研摘要

截至 2026 年初，MCP 调研生态已相当成熟。最优组合是：Brave Search（免费通用搜索）+ Tavily 或 Exa（AI 优化搜索带内容提取）+ Firecrawl（深度爬取）+ Context7 或替代品（库文档）。Claude Code 内置 WebSearch 满足约 80% 的临时需求，但缺乏批量操作和内容预提取能力。

---

## 各工具详细分析

### 1. Context7 (MCP) — 库/API 文档

**最强项**: 解决"训练数据过时"问题——动态注入最新版本库文档，避免 Claude 使用废弃 API。被视为最成熟的文档 MCP server。

**弱点**:
- 2026 年 1 月免费配额从 ~6,000 降至 1,000 次/月，加了 60 次/小时速率限制
- 用户反映第一周就用完配额
- 只能查已索引的库，不是搜索引擎

**定价**: 免费 1,000 次/月（2026 年 1 月后缩减）

**速度**: 快（索引查询）

**MCP server**: 是（官方，Upstash 出品）

**社区评价**: 高度认可，但 2026 年 1 月配额削减促使很多用户迁移到替代品

---

### 2. Tavily (MCP) — AI 优化搜索 + 提取

**最强项**:
- 专为 AI agent / RAG 管道设计
- 结果预清洗，直接适合 LLM 消费
- 与 LangChain/LlamaIndex/CrewAI 深度集成
- 1,000 免费积分/月——AI 搜索 API 中最慷慨的免费层
- 官方 MCP server，production-grade

**弱点**:
- 没有自己的搜索索引（聚合器）
- 无结构化提取（需后处理，不如 Firecrawl）
- MCP benchmark 成功率仅 45%（Firecrawl 83%）
- 积分不滚存

**定价**:
- 免费: 1,000 积分/月
- PAYG: $0.008/积分（$8 CPM）
- $30/月 4,000 积分起

**速度**: MCP 任务平均 41.3 秒

**MCP server**: 是（官方）

---

### 3. Exa (MCP) — 语义搜索 + 代码上下文

**最强项**:
- 语义搜索（神经嵌入，理解查询意图）
- 复杂检索准确率 81% vs Tavily 71%（独立 benchmark）
- 比 Tavily 快 2-3 倍
- 人员搜索（10 亿+ LinkedIn profile）
- 公司搜索、代码搜索
- 返回预提取内容（无需二次 fetch）
- 有效过滤 SEO 垃圾

**弱点**:
- 简单关键词查询比 Brave 贵
- 配置对新手有难度
- 高峰时段偶有限速（用户反映）
- 不同 benchmark 速度数据矛盾（任务类型相关）

**定价**:
- 免费: $10 credits（不需信用卡）
- Starter: $49/月 8,000 credits
- Pro: $449/月 100,000 credits

**速度**: 语义查询比 Tavily 快 2-3 倍；复杂 MCP 任务约 231 秒

**MCP server**: 是（官方）

---

### 4. Brave Search (MCP) — 隐私优先网络搜索

**最强项**:
- 100% 独立索引（2023 年 4 月起不依赖 Bing/Google）
- 每天处理 5000 万次搜索，95% 请求 <1 秒
- 2,000 次免费/月——通用搜索最好的免费层
- Anthropic 官方推荐用于 Claude Desktop
- 隐私安全，适合医疗/法律/金融

**弱点**:
- 技术文档检索"缺乏深度和时效性"
- 索引覆盖面小于 Google/Bing
- 不是 AI 原生（返回传统搜索结果，非预提取内容）

**定价**:
- 免费: 2,000 次/月
- 付费: $5-9/1,000 请求

**速度**: 最快，<15 秒

**MCP server**: 是（官方，Anthropic 推荐）

---

### 5. Perplexity (API + MCP)

**最强项**:
- 返回综合答案而非原始 URL——减少 Claude 的处理负担
- Deep Research 模式（sonar-deep-research 模型）
- 时效性过滤——对快速变化的技术话题关键
- 支持领域过滤和学术源优先
- 官方 MCP server + 多个高质量社区版本

**弱点**:
- 每次查询成本比简单搜索 API 高
- 综合可能引入幻觉（比原始搜索控制少）
- 不适合结构化数据提取
- 简单查询用它大材小用

**定价**: ~$1/百万 tokens（因模型不同而异）；Deep Research 模型更贵

**速度**: sonar 快，sonar-deep-research 慢（设计如此）

**MCP server**: 是（官方 `perplexityai/modelcontextprotocol`）

---

### 6. Firecrawl (MCP) — 网页爬取/抓取

**最强项**:
- 最快的 MCP 网页提取：平均 7 秒
- Benchmark 准确率 83%（Tavily 45%）
- Schema 结构化提取：电商准确率 94%
- 8 个工具：scrape/crawl/map/search/extract/批量/深度研究/AI agent
- `firecrawl_deep_research`：从自然语言 prompt 自主多步研究
- 规模化最便宜：100k 操作 $83/月 vs Tavily $800

**弱点**:
- 不是搜索引擎——需配合 Brave/Tavily/Exa 发现 URL
- 免费层仅 500 积分（~50 次爬取或 25 次搜索）
- 简单搜索比 Brave 贵

**定价**:
- 免费: 500 积分
- Hobby: $16/月 3,000 积分
- Standard: $83/月

**速度**: 最快（7 秒平均）

**MCP server**: 是（官方）

---

### 7. Claude Code 内置 WebSearch + WebFetch

**最强项**:
- 零配置零成本，所有 Claude Code 用户可用
- 满足约 80% 的日常开发调研需求
- 无需 API 密钥

**弱点**:
- 两步操作（搜索 + 抓取分开）
- 只返回 Markdown，无原始 HTML
- 每次搜索结果数量有限
- 不支持批量操作
- AWS Bedrock / Google Vertex 不可用
- WebFetch 100KB 限制，15 分钟缓存

**定价**: Claude Code 内含，无额外费用

**MCP server**: 无（内置）

---

### 8. Baidu Search MCP — 中文内容

**最强项**:
- 访问中文网页内容的唯一实用选择
- 部分实现支持百度文心 AI 搜索

**弱点**:
- 全是社区实现，无官方 Baidu MCP
- 维护质量参差不齐
- HuChundong 版用浏览器自动化（慢，脆弱）

**主要实现**:
- `Evilran/baidu-mcp-server`：TypeScript，高级限速，LLM 友好输出（推荐）
- `appleinmusic/baidu-search-mcp`：使用百度文心 API，支持深度搜索

**定价**: 一般免费（需百度 API key）

**MCP server**: 是（多个社区实现）

| 工具 | 成功率 | 平均速度 |
|------|--------|----------|

| Nimble | 93% | ~25s |

## 2025-2026 新兴 MCP 工具

### 搜索聚合
- **mcp-omnisearch**：单一 MCP 整合 Tavily + Brave + Kagi + Perplexity + Jina + Exa + Firecrawl

### 深度研究 Agent
- **GPT Researcher MCP**：完整深度研究 agent，输出带引用报告
- **Parallel Task MCP**：异步研究任务（首创 async MCP 模式）
- **Deep Research MCP Server**（ssdeanx）：Gemini 2.5 Flash 驱动

### 内容提取
- **Jina Reader MCP**（官方）：任意 URL 转 Markdown，10M 免费 tokens，含 `search_arxiv`
- **Linkup MCP**：搜索 + 页面抓取，支持 Deep Search 模式
- **Fetch MCP**：轻量 URL 转内容，Firecrawl 的轻量替代

### 专项研究
- **Reddit Research MCP**：2 万+ subreddit 语义搜索
- **BioMCP**：PubMed + ClinicalTrials.gov + MyVariant
- **DeepWiki**：GitHub 仓库转 AI 增强 Wiki

## Context7 替代品（2026 年 1 月配额削减后）

| 工具 | 免费层 | 核心差异 |
|------|--------|----------|
| Context（Neuledge） | 无限（本地）| 完全离线，SQLite，无速率限制 |
| Nia（YC 投资）| 免费 + $14.99/月 | 幻觉率低 11% |
| Deepcon | Freemium | 基准准确率 90% vs Context7 的 65% |
| Docfork | 1,000 次/月 | 9,000+ 库，"Cabinets" 上下文隔离 |
| GitMCP | 免费无限 | 零配置，任意 GitHub 公共仓库 |
| Ref Tools | 免费 + $9/月 | Session 感知，5K token 上限 |

**社区推荐替换组合**: GitMCP（零配置免费）+ Docfork（广覆盖）

## Power User 推荐组合

### 最小可行调研 Stack
1. Brave Search — 网页发现
2. Fetch — 已知 URL 内容提取
3. Memory MCP — 跨会话持久知识
4. Filesystem — 本地保存

### 完整调研 Stack
1. Brave Search — 免费基础搜索
2. Exa — 语义/复杂查询
3. Firecrawl — 深度爬取已知 URL
4. Perplexity — 复杂问题综合答案
5. Context7 / GitMCP — 编码时查库文档
6. Sequential Thinking MCP — 多步推理

### 中文内容研究
在上述 stack 基础上加 Baidu MCP（推荐 Evilran 实现）

### 全免费 Stack
1. Claude Code 内置 WebSearch + WebFetch
2. Brave Search MCP（2,000/月）
3. Tavily MCP（1,000/月）
4. GitMCP（无限免费）
5. Jina Reader MCP（10M tokens 免费）

## 使用场景决策矩阵

| 场景 | 推荐工具 |
|------|----------|
| 临时快速查询 | 内置 WebSearch |

| 时事/新闻 | Perplexity 或 Brave |
| 深度语义研究 | Exa |
| 爬取已知 URL | Firecrawl 或 Jina Reader |

| 需要综合答案 | Perplexity |
| 大规模批量提取 | Firecrawl（规模最便宜）|
| 复杂多步研究 | GPT Researcher MCP 或 Parallel Task MCP |
