# 调研报告: 跨领域 AI 辅助调研策略（2025-2026 最佳实践）

**日期**: 2026-03-11
**任务**: 系统整理不同类型来源（API文档、开源项目、学术论文、中文内容、技术社区讨论）的最佳调研工具和策略，包括 MCP 生态覆盖情况和中英双语研究建议。

---

## 调研摘要

2025-2026 年 AI 辅助调研已进入 MCP 工具化时代，几乎每类信息源都有对应的 MCP Server。核心原则是：**用对工具比用好工具更重要**——时效性内容用 Perplexity/Tavily，语义相似性发现用 Exa，API 文档用 Context7，学术论文用 Semantic Scholar MCP，中文内容用专项 MCP（Zhihu、PulseCN、Open-WebSearch 含 CSDN/掘金）。单一工具无法覆盖所有场景，组合策略是主流。

---

## 1. 最新 API/库文档获取

### 工具对比

| 工具 | 原理 | 适合场景 | 局限 |
|------|------|----------|------|
| **Context7 MCP** | 拉取官方文档，版本化索引，9000+ 库 | 主力工具，LLM 内使用 | 不覆盖所有库 |
| **Docs MCP Server** (arabold) | 自托管，支持网站/GitHub/npm/PyPI | 私有文档、不在 Context7 的库 | 需自行部署 |
| **GitHub README** | 直接读源文件 | 最新但非结构化 | 需要 gh CLI 或 WebFetch |
| **官方网站** | WebFetch 抓取 | 无 MCP 的文档 | 需要知道 URL |

### 核心策略

1. **首选 Context7**：在 prompt 中加 "use context7"，自动获取版本正确的文档片段
2. **Context7 没有时**：`gh repo view {owner/repo} --json` + README，或 WebFetch 官方文档
3. **版本敏感时**：Context7 支持在 prompt 中指定版本（"use context7, library v2.1"）
4. **私有/内部库**：用 Docs MCP Server（self-hosted）建立本地文档索引

### 常见陷阱
- 不要依赖 LLM 训练数据中的 API 知识（知识截止日期问题）
- Context7 文档抓取可能有轻微延迟（最新 RC/Alpha 版本可能不全）
- GitHub README 不代表稳定 API，注意看 CHANGELOG

### MCP 状态
- Context7 MCP: 官方支持，Smithery 可安装 ✅
- Docs MCP Server: 开源自托管 ✅

## 2. 开源项目发现与评估

### 工具对比

| 工具 | 强项 | 适用场景 |
|------|------|----------|
| **gh CLI** (GitHub) | 精确过滤（stars/language/date）| 找代码实现 |
| **Exa AI** | 语义相似性搜索 | 概念性发现，"找类似 X 的项目" |
| **HuggingFace Hub** | 模型/数据集排行榜 | AI/ML 领域专项 |
| **awesome-* 列表** | 人工筛选，质量高 | 特定领域入门 |

### 核心策略

**GitHub 发现流程**:
```
gh search repos --topic={topic} --stars=">100" --language={lang} --sort=updated
gh repo view {owner/repo}  # 读 README + Stars + last commit
```

**HuggingFace 模型评估**:
1. 看 Community Leaderboard（任务专项排行榜）
2. 看 Model Card（能力声明 + 局限性）
3. 从 3-5 个候选开始，构建自己的小评估集（比公开 benchmark 更有效）
4. 权衡：准确率 vs 推理速度 vs 内存占用

**Exa 发现策略**:
- 适合语义查询："find projects similar to LangChain but lighter weight"
- 不适合：精确版本、星数过滤

### 常见陷阱
- Stars 数量不代表代码质量（营销驱动的 stars 很多）
- 最后 commit 时间比 stars 更能反映维护状态
- HuggingFace 排行榜 benchmark 和实际用例差距可能很大

### MCP 状态
- GitHub 官方 MCP: 存在 ✅
- HuggingFace 官方 MCP: 存在（huggingface.co/settings/mcp）✅
- Exa MCP: 存在 ✅

## 3. 学术论文与研究

### 工具生态

| 工具 | 类型 | 强项 |
|------|------|------|
| **Semantic Scholar** | 免费，AI增强 | TLDR摘要、引用图谱、开放API |
| **Elicit** | 付费 | 结构化文献综述，表格对比 |
| **Connected Papers** | 可视化 | 从一篇论文发现关联文献网络 |
| **arXiv** | 预印本 | 最新研究（未经同行评审） |
| **PubMed** | 医学/生物 | 领域专项 |

### 核心策略（Semantic Scholar 最优工作流）

1. **向后追溯**：用 "Highly Influential Citations" 找基础性论文
2. **向前延伸**：用 "Cited By" 找引用该论文的最新工作
3. **横向发现**：用 "Related Papers" 找同一领域其他角度
4. **验证**：跨 Scholar/Scopus/Web of Science 核对（避免遗漏）

**Semantic Scholar API 最佳实践**:
- 每次请求只要必要字段（避免拖慢速度）
- 大批量用 bulk/batch 版本端点
- 免费 API Key（1 RPS），大量数据考虑下载数据集本地查询

### 工具组合建议
- 探索阶段：Semantic Scholar + Connected Papers（可视化网络）
- 深度阅读：Elicit（表格化对比多篇论文的方法/结论）
- 最新预印本：arXiv 直接搜索

### 常见陷阱
- arXiv 论文未经同行评审，结论需谨慎
- 引用数高不代表结论正确（热门错误论文也存在）
- Elicit 付费但适合系统性文献综述

### MCP 状态
- Semantic Scholar MCP (FujishigeTemma): 支持论文搜索/详情/作者/引用导出 ✅
- Academia MCP (IlyaGusev): 覆盖 arXiv + ACL + HuggingFace Datasets + Semantic Scholar ✅
- arXiv MCP Server (blazickjp): 专项搜索和内容访问 ✅

## 4. 中文内容调研

### 平台特性

| 平台 | 内容类型 | 质量 | 最适合 |
|------|----------|------|--------|
| **知乎** | Q&A、专栏 | 中高 | 概念理解、行业洞察 |
| **掘金 (Juejin)** | 技术博客 | 中高 | 前端/全栈技术实践 |
| **CSDN** | 技术博客 | 参差不齐 | 关键词搜索，大量但需筛选 |
| **V2EX** | 社区讨论 | 中 | 开发者真实反馈、工具讨论 |
| **Bilibili** | 视频 | 中 | 教程、演示 |
| **微信公众号** | 文章 | 中高 | 行业分析、深度报告 |
| **36kr** | 科技新闻 | 高 | 行业动态、创投 |

### 搜索策略

**Baidu**（占中国搜索市场 46%）:
- 搜中文技术问题时，百度比 Google 更能找到 CSDN/知乎结果
- 使用 `site:zhihu.com` 或 `site:juejin.cn` 精准过滤

**Google 搜索中文内容**:
```
site:zhihu.com {关键词}
site:juejin.cn {关键词}
{关键词} site:v2ex.com
```

**专项 MCP**（最新可用）:
- **Open-WebSearch MCP**: 支持 Bing/Baidu/Brave/DuckDuckGo/Exa，专项提取 CSDN/知乎/LinuxDo 内容
- **WebSearchMCP**: 多引擎支持 Bing/Baidu/GitHub/掘金/CSDN
- **Zhihu MCP**: 知乎专项（主要用于发布，搜索功能有限）
- **Pulse CN MCP**: 实时中国互联网热点（微博/知乎/抖音/B站/豆瓣/头条/36kr）
- **Chinese Trends Hub MCP**: 类似 Pulse CN

### 常见陷阱
- CSDN 内容质量参差，存在大量复制粘贴
- 知乎答案有时候过时（看回答日期）
- 微信文章无法直接搜索到全文（需微信内搜索或公众号导航站）

### MCP 状态
- Open-WebSearch MCP: 含百度 + CSDN/知乎提取 ✅
- Pulse CN MCP: 中国热点追踪 ✅
- Xiaohongshu MCP: 小红书专项 ✅

## 5. 技术社区讨论

### 平台状态（2025-2026）

| 平台 | 2025 状态 | 最适合 |
|------|-----------|--------|
| **Stack Overflow** | 问题量跌 95%，成"博物馆" | 经典问题查历史答案 |
| **Reddit** (/r/programming 等) | 蓬勃发展，讨论活跃 | 工具对比、社区意见、真实体验 |
| **Hacker News** | 活跃，技术深度高 | 新技术讨论、创业话题 |
| **Discord** | 项目官方社区 | 最新问题、作者直接回答 |
| **V2EX** | 中文开发者社区 | 国内开发者视角 |

### 搜索技巧

**Hacker News (HN)**:
- 官方搜索：`hn.algolia.com`（Algolia 驱动，每2分钟更新）
- API 可用：`https://hn.algolia.com/api`（CORS 友好，返回 JSON）
- 过滤：按 story/comment 类型，按日期排序

**Reddit**:
- Google: `site:reddit.com/r/MachineLearning {关键词}`
- Pushshift 历史数据（注意：2023年后受限）
- Reddit 原生搜索改进了，但 Google 仍更强

**Stack Overflow**:
- `site:stackoverflow.com {关键词}` via Google 最有效
- 仍有大量有价值的历史答案（尤其是 2010-2020 的问题）
- 新问题几乎没人回答了，AI 替代了新问题功能

**Discord**:
- 没有通用搜索 API，需在各项目官方 Discord 内搜索
- 很多项目在 Discord 上有 #announcements 和 #faq

### MCP 状态
- HN MCP: 有非官方实现，Algolia API 本身可 WebFetch ✅
- Reddit MCP: 官方存在（但 API 2023 年后付费限制） ⚠️
- Discord: 无公开 MCP，需在 Discord 内手动搜索 ❌

## 6. 时效性内容 vs 常青内容策略

### 决策矩阵

| 问题类型 | 推荐工具 | 原因 |
|---------|---------|------|
| "X 的最新版本是什么" | Perplexity / Tavily | 速度快，实时数据 |
| "X 库有没有安全漏洞" | Perplexity（新闻抓取） | 时效敏感 |
| "什么架构模式适合 Y" | Exa（语义搜索）+ Semantic Scholar | 概念性，需深度理解 |
| "如何用 Z 实现功能 A" | Context7（文档）+ Reddit/HN | 实践性，社区验证 |
| "X 和 Y 哪个更好" | Reddit（真实意见）+ HN | 社区讨论，多角度 |
| "X 领域最新研究进展" | arXiv + Semantic Scholar | 学术，需引用追踪 |

### 工具速度/深度权衡

```
速度: Perplexity > Tavily > Exa > Semantic Scholar
深度: Semantic Scholar > Exa > Perplexity > Tavily
语义理解: Exa > Perplexity ≈ Semantic Scholar > Tavily
中文内容: Open-WebSearch(Baidu) > Perplexity > 其他
```

### 推荐组合策略（"Research Stack"）

**快速调研（< 30分钟）**:
1. Perplexity → 快速背景建立
2. Context7 → API/库具体用法
3. Reddit/HN 搜索 → 社区真实评价

**深度调研（> 2小时）**:
1. Semantic Scholar → 学术基础
2. Connected Papers → 文献网络可视化
3. Exa → 语义发现更多相关项目/文章
4. GitHub (gh CLI) → 代码实现
5. HuggingFace → 模型/数据集
6. 中文：Open-WebSearch(Baidu+CSDN+知乎) 补充国内视角

## 7. 中英双语调研策略

### 核心洞察

研究显示 LLM 在不同语言下表现出不同的文化倾向。中英双语调研的价值在于：
- **英文**：更多前沿研究、开源项目、学术论文、国际社区讨论
- **中文**：国内实践经验、本土化方案、中国用户视角、部分领先的工程实践

### 实操建议

1. **搜索词翻译**：用 Claude 把关键词翻成中文再搜，避免漏掉中文专有术语
   - 例："RAG" → "检索增强生成"，"fine-tuning" → "微调"
2. **中文先行，英文验证**：快速中文搜获取概念，英文搜验证准确性
3. **英文先行，中文补充**：英文搜技术方案，中文搜国内实践案例
4. **平台分工**:
   - 英文：arXiv/HuggingFace/GitHub/HN/Reddit
   - 中文：知乎（概念洞察）/掘金（前端实践）/V2EX（开发者反馈）

## 推荐方案（最优 Research Stack 组合）

### 日常技术调研（推荐配置）

| 需求 | 工具 | MCP |
|------|------|-----|
| API 文档 | Context7 | context7-mcp ✅ |
| OSS 发现 | gh CLI + Exa | github-mcp ✅ / exa-mcp ✅ |
| 学术论文 | Semantic Scholar | semantic-scholar-mcp ✅ |
| 最新动态 | Perplexity / Tavily | perplexity-mcp ✅ |
| 中文内容 | Open-WebSearch(Baidu) | open-websearch-mcp ✅ |
| 社区讨论 | HN Algolia + Reddit | hn.algolia.com API |
| AI/ML 模型 | HuggingFace Hub | huggingface-mcp ✅ |

### 实施建议

**关键步骤**:
1. 安装 Context7 MCP 和 HuggingFace MCP（最高 ROI）
2. 配置 Open-WebSearch MCP 覆盖中文平台
3. 配置 Semantic Scholar MCP 用于学术调研
4. 建立 "时效性 vs 常青" 判断习惯，选择对应工具

**风险点**:
- Reddit API 2023 年后商业化，部分 MCP 受限 — 缓解：用 Google `site:reddit.com` 替代
- Discord 无公开 API — 缓解：手动搜索，或在对应项目社区注册
- Context7 覆盖率 9000+ 库但非全部 — 缓解：fallback 到 Docs MCP Server 或 WebFetch

**依赖项**:
- MCP 配置：Claude Desktop 或 Cursor 或 VS Code Copilot
- API Keys：Exa（有免费额度）、Semantic Scholar（免费注册）、Perplexity（付费）
