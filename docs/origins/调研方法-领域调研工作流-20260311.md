# 调研报告: 六大领域调研工作流对比分析

**日期**: 2026-03-11
**任务**: 研究六个领域调研工作流的实质差异——步骤、顺序、领域独特性

---

## 调研摘要

不同领域的调研工作流在四个维度上根本分叉：问题确定性、证据标准、完成判断权归属、犯错代价。技术工程领域用"Spike+ADR"追求"够有信心可以提交"；学术领域用"PRISMA+双盲审"追求"方法可重复"；商业领域用"TAM/SAM/SOM+竞争情报"追求"可行动的市场信心"；产品UX用"双钻模型+情境观察"追求"用户实际行为洞察"；AI/ML用"自建eval数据集+影子部署"追求"生产环境验证"；教育领域用"行动研究+证据层级"追求"本地适配的实践改进"。

## 关键发现（精简版）

### 各领域独有步骤

| 领域 | 独有步骤 | 存在原因 |
|------|----------|----------|
| 学术 | 预注册（pre-registration） | 防止事后假设（HARKing）和发表偏倚 |
| 学术 | 双盲独立筛选 | 控制选择偏倚 |
| 工程 | Spike（一次性代码） | 低成本获取知识再提交 |
| 工程 | ADR作为决策产物 | 异步组织决策、防止重复讨论 |
| 商业 | TAM/SAM/SOM | 投资和资源分配决策 |
| 商业 | 持续竞争情报监测 | 市场是动态对抗环境 |
| UX | 情境观察（用户真实环境） | 可用性取决于真实环境不是实验室 |
| UX | 饱和度作为停止标准 | 定性研究，无法预设样本量 |
| AI/ML | 自建eval数据集 | 基准测试数据≠生产数据 |
| AI/ML | 影子部署 | ML系统生产行为不可预测 |
| AI/ML | 分布漂移监测 | 模型随数据变化退化，软件不会 |
| 教育 | 实施保真度评估 | 教学干预因实施者不同差异大 |
| 教育 | 行动研究 | 从业者在自己教室中做研究 |

### "完成"的定义差异

- **工程**：决策已做出、记录在案、不会被重新讨论（除非有新证据）
- **学术**：同行评审发表。未发表的手稿不算"完成"
- **商业**：决策者信心足够可以行动（而非所有问题都回答了）
- **产品UX**：定性——达到饱和；定量——达到统计显著性
- **AI/ML（工程师）**：模型已部署且生产指标优于基线
- **教育（从业者）**：学生表现更好 + 适配足够成熟可以分享

### 多领域并行调研要点

1. **以首要决策者的领域为"完成"标准**
2. **各领域并行运行，设置集成时间点**
3. **明确标注证据来源领域**（商业假设不是同行评审，学术发现不是市场验证）
4. **时间有限时所有领域都用轻量版**

### AI最有价值的步骤

- 信息检索与合成（全领域）
- 模板生成（ADR/访谈提纲/PICO）
- 大批量文档的一轮筛选
- 转录和文档的主题提取
- 报告撰写

### AI无法替代的步骤

- 情境观察（无法在现场）
- 证据质量判断（需要专业判断力）
- 利益相关者动态（组织政治）
- 真正新颖的假设生成
- 基于关系的知识传递
