# W2: 用户洞察（"用户在抱怨什么"）

**目标**：发现用户真实痛点和需求
**典型耗时**：L2 = 20-40min

## 步骤

### SCOPE
定义目标用户群 + 想了解的维度。

### LISTEN（多渠道采集，并行）
- 社区: Reddit/HN "frustrating about X" / 知乎 "X 有什么问题"
- 评论: App Store/G2/Capterra 差评提取
- 竞品反馈: 竞品的 GitHub Issues / 用户论坛
- 国内: V2EX/掘金评论区

### CLUSTER
痛点归类（频率排序）：

| 痛点 | 提及频率 | 严重度 | 现有解决方案 | 来源 |
|------|---------|--------|------------|------|

### VALIDATE
交叉验证：同一痛点是否多渠道出现。

### INSIGHT
提炼为 "用户需要 [X]，因为 [Y]" 格式。

### PERSIST
按持久化规则写报告 + 更新 INDEX.md。
