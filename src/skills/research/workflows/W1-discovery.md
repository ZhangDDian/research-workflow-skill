# W1: 方案发现（"找轮子"）

**目标**：快速回答"市面上有没有现成方案？"
**典型耗时**：L2 = 15-30min, L3 = 1-2h

## 步骤

### SCOPE
定义需求边界 + 必须/可选功能清单。

### SCAN（三路并行搜索）
- GitHub: `gh search repos --topic=X --stars=">50" --sort=updated`
- 产品搜索: Exa/Tavily "X alternatives" / "tools for X"
- 社区: Reddit/HN/知乎 "推荐一个X" / "X 哪个好用"

### FILTER
按约束过滤：
- 语言/技术栈匹配
- 许可证（MIT/Apache/BSD）
- 维护状态（最近更新 < 6个月）
- 社区活跃度（Stars > 100, Issues 关闭率 > 50%）

### COMPARE
Top 3-5 候选方案对比表：

| 方案 | 功能匹配度 | 维护状态 | 社区 | 许可证 | 集成成本 |
|------|-----------|---------|------|--------|---------|

### RECOMMEND
推荐 + 复用策略标签：
- 🟢 直接用（功能匹配、维护良好）
- 🟡 借鉴（核心思路可用，需适配）
- 🔵 参考（设计思路有价值，自己实现）

### PERSIST
按持久化规则写报告 + 更新 INDEX.md。
